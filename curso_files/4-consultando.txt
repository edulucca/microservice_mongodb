1 - As consultas possuem 5 níveis:
1.1 find - define o filtro (semelhante ao WHERE):
{name: "Eduardo de Lucca"}

1.2 Project - define quais campos devem ser retornados na consulta (semelhante ao SELECT). 1 = true / 0 = false.
{name:1,email:1}

1.3 Sort - define com base em qual campo deve ordenar a consulta. (semelhante ao ORDER BY). 1 = crescente  / -1 = decrescente.
{name: 1}

1.4 Collation - define a language da consulta e se deve considerar o case sensitive (strength:2 = no case sensitive)
{locale:"pt", strength:2}

1.5 Index Hint - o hint aconselha o banco a usar um índice específico na consulta, em caso em que a escolha automática do índice não é ideal. O hint só funciona caso o index esteja devidamente criado para o campo selecionado. Por padrão, o campo _id é um index. 1 = true.
{_id:1}


Opcionais:
1 - Max Time MS: define o tempo limite de consulta
2 - Skip: pula um numero especifico de objetos no topo da consulta
3 - Limit: define o numero máximo de linhas da consulta


Em mongosh:

1 - busca comum com filtro por campo:
db.users.find({name:"Eduardo de Lucca"})

2 - para usar o collation:
db.users.find({name:"eduardo de lucca"}).collation({locale:"pt", strength:2})

3 - para usar o Project (projection):
db.users.find({name:"eduardo de lucca"}).collation({locale:"pt", strength:2}).projection({name:1, email:1})

4 - para usar o sort:
db.users.find().sort({name:1})
